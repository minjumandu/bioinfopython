## DNA, RNA, 단백질의 관계

DNA는 유전 정보를 저장하는 분자입니다. RNA는 그 정보를 전달하거나 활용하는 과정에 관여하며, 단백질은 실제로 세포 안에서 다양한 기능을 수행하는 분자입니다. 매우 단순하게 말하면, DNA는 설계도이고, RNA는 설계도를 옮겨 적은 작업 지시서이며, 단백질은 실제로 만들어진 결과물이라고 비유할 수 있습니다.

생물정보학에서는 이 흐름을 데이터의 관점에서 바라봅니다. DNA 서열은 문자 데이터이고, RNA 서열도 또 다른 문자 데이터이며, 단백질 서열 역시 아미노산 문자들의 나열로 표현할 수 있습니다. 즉, 서로 다른 생물학적 분자도 컴퓨터 안에서는 모두 일정한 규칙을 가진 정보로 다룰 수 있습니다.

## 중심원리(Central Dogma) 이해하기

중심원리는 일반적으로 다음과 같이 표현됩니다.

DNA → RNA → Protein

이 흐름은 정보의 저장, 전사, 번역이라는 세 단계로 이해할 수 있습니다.

1. DNA는 유전 정보를 저장한다.
2. DNA 정보의 일부는 RNA로 전사된다.
3. RNA 정보는 단백질 서열로 번역된다.

물론 실제 생명현상은 이보다 더 복잡하지만, 중심원리는 생물정보학의 기본 개념을 이해하는 데 매우 중요한 출발점입니다.

앞서 배운 전사 함수는 이 개념의 아주 단순한 코드 표현이라고 볼 수 있습니다.

```python
def transcribe(dna_seq):
    return dna_seq.upper().replace("T", "U")


dna = "ATGCGTAC"
rna = transcribe(dna)

print("DNA:", dna)
print("RNA:", rna)
```

이 예제는 DNA가 RNA로 바뀌는 과정을 아주 단순하게 흉내 냅니다. 생물학을 코드로 바꿔 보는 이런 경험이 바로 생물정보학적 사고의 출발입니다.

## 서열에서 정보의 흐름 읽기

생물정보학은 서열을 단순한 문자 집합으로 보는 데서 멈추지 않습니다. 중요한 것은 그 서열 안에 어떤 정보가 숨어 있는지 질문하는 것입니다.

예를 들어 다음과 같은 질문을 떠올릴 수 있습니다.

- 이 DNA 서열의 길이는 얼마인가?
- GC 함량은 얼마나 되는가?
- RNA로 전사하면 어떤 형태가 되는가?
- 상보 서열은 무엇인가?
- 특정 유전자와 관련된 패턴이 보이는가?

우리는 이미 이 책에서 이러한 질문을 파이썬 코드로 바꾸는 방법을 배웠습니다. 즉, 생물정보학은 생물학적 질문을 계산 가능한 형태로 다시 쓰는 작업입니다.

## 생물학 개념을 코드로 바꾸는 사고법

이 책 전체를 관통하는 중요한 생각은 다음과 같습니다.

1. 먼저 생물학적 질문을 분명히 한다.
2. 그 질문을 데이터의 형태로 바꾼다.
3. 적절한 파이썬 도구로 분석한다.
4. 결과를 표와 그래프로 해석한다.

예를 들어 "이 처리 조건이 유전자 발현을 증가시키는가?"라는 질문은 표 데이터와 평균 비교, 시각화 문제로 바뀔 수 있습니다. "이 서열은 정상적인 DNA인가?"라는 질문은 문자열 검사와 조건문 문제로 바뀔 수 있습니다. "FASTA 파일에서 서열만 읽어 오고 싶은가?"라는 질문은 파일 입출력과 파싱 문제로 바뀔 수 있습니다.

이런 사고법을 익히면, 앞으로 더 복잡한 생물정보학 문제를 만나더라도 스스로 분석의 첫 단계를 설계할 수 있게 됩니다.

## 미니 프로젝트: 작은 생물정보학 분석 리포트 만들기

이제 책의 마지막으로, 지금까지 배운 내용을 활용하는 작은 프로젝트를 제안합니다. 하나의 FASTA 파일이나 짧은 유전자 발현 데이터를 준비한 뒤, 다음 과정을 수행해 보십시오.

1. 데이터를 읽어 온다.
2. 데이터가 올바른지 검사한다.
3. 길이, GC 함량, 또는 평균 발현값 같은 기본 정보를 계산한다.
4. 결과를 간단한 표나 그래프로 정리한다.
5. 마지막으로 짧은 해석 문장을 작성한다.

예를 들어 FASTA 기반 미니 프로젝트는 다음과 같이 시작할 수 있습니다.

```python
def parse_fasta(file_name):
    sequence = ""
    with open(file_name, "r") as f:
        for line in f:
            if not line.startswith(">"):
                sequence += line.strip().upper()
    return sequence


def calculate_gc_content(seq):
    gc_count = seq.count("G") + seq.count("C")
    return (gc_count / len(seq)) * 100


dna = parse_fasta("sample.fasta")

print("서열 길이:", len(dna))
print(f"GC 함량: {calculate_gc_content(dna):.2f}%")
print("RNA 서열:", dna.replace("T", "U"))
```

이 프로젝트는 매우 작지만, 실제 생물정보학 작업의 기본 구조를 모두 담고 있습니다. 데이터 읽기, 검증, 계산, 해석이 하나의 흐름으로 연결되기 때문입니다.
