# 3장 조건문과 반복문으로 DNA 서열 검사하기

2장에서 우리는 DNA 서열을 문자열로 저장하고, 리스트와 숫자 자료형으로 다루는 법을 배웠습니다. 이제 한 걸음 더 나아가 보겠습니다. 데이터를 저장하는 것만으로는 분석이 끝나지 않습니다. 실제 생물정보학에서는 "이 서열이 올바른가?", "특정 염기는 몇 번 나타나는가?", "서열 전체를 순서대로 살펴보려면 어떻게 해야 하는가?"와 같은 질문을 계속 던지게 됩니다.

이때 필요한 것이 바로 제어문입니다. 조건문은 어떤 상황에서 무엇을 할지 판단하게 해 주고, 반복문은 같은 작업을 여러 번 수행하게 해 줍니다. 생물정보학에서 이 둘은 매우 중요합니다. DNA 서열의 각 문자를 하나씩 검사하고, 잘못된 문자를 찾고, 특정 염기의 개수를 세는 일은 모두 조건문과 반복문으로 이루어집니다.

그림 3-1은 이 장의 핵심을 보여 줍니다. DNA 서열을 처음부터 끝까지 순회하면서, 각 염기가 올바른지 조건으로 검사하고, 필요한 값을 누적하면 기초적인 생물정보학 분석이 가능합니다.

## 학습 목표

이 장을 읽고 나면 다음 내용을 설명하고 실습할 수 있어야 합니다.

- 조건문의 기본 구조를 이해할 수 있다.
- 반복문을 이용해 DNA 서열을 처음부터 끝까지 검사할 수 있다.
- 유효하지 않은 문자를 포함한 서열을 판별할 수 있다.
- 특정 염기의 개수를 세는 간단한 프로그램을 작성할 수 있다.

## DNA 유효성 검사의 필요성

생물정보학에서 다루는 데이터는 항상 완벽하지 않습니다. 실험 과정에서 오류가 생길 수도 있고, 파일을 잘못 복사할 수도 있으며, 사람이 직접 입력하는 과정에서 오타가 들어갈 수도 있습니다. 예를 들어 DNA 서열에는 원칙적으로 A, T, G, C만 등장해야 하는데, 만약 `X`나 `Z` 같은 문자가 들어 있다면 그 서열은 정상적인 DNA 서열로 보기 어렵습니다.

따라서 분석의 첫 단계는 데이터를 믿기 전에 검사하는 것입니다. 실제 연구에서도 데이터 정제와 유효성 검사는 매우 중요한 과정입니다. 잘못된 데이터를 그대로 분석하면 결과도 잘못될 수 있기 때문입니다.

## `if` 문으로 잘못된 염기 찾기

조건문(`if`)은 "만약 어떤 조건이 참이라면 이 코드를 실행하라"는 뜻을 담고 있습니다. DNA 유효성 검사에서는 어떤 문자가 A, T, G, C에 속하는지 확인하는 데 조건문을 사용할 수 있습니다.

```python
base = "A"

if base in "ATGC":
    print("올바른 DNA 염기입니다.")
```

위 코드는 `base`가 `"ATGC"` 안에 포함되어 있는지 검사합니다. 만약 참이라면 "올바른 DNA 염기입니다."라는 문장이 출력됩니다.

반대로 잘못된 염기를 찾고 싶다면 다음과 같이 쓸 수 있습니다.

```python
base = "X"

if base not in "ATGC":
    print("잘못된 DNA 염기입니다.")
```

이처럼 조건문은 데이터가 올바른지 판단하는 가장 기본적인 도구입니다.

## `for` 문으로 서열 처음부터 끝까지 순회하기

DNA 서열은 보통 한 글자만 검사하는 것이 아니라, 처음부터 끝까지 모든 문자를 확인해야 합니다. 이때 사용하는 것이 반복문(`for`)입니다.

```python
dna = "ATGCGTAC"

for base in dna:
    print(base)
```

이 코드는 `dna` 문자열 안의 각 문자를 하나씩 꺼내어 `base`에 담고 출력합니다. 결과적으로 서열의 모든 염기가 한 줄씩 차례로 출력됩니다.

반복문은 "문자열 전체를 하나하나 훑어본다"는 감각을 익히는 데 중요합니다. 실제 생물정보학에서는 서열 전체를 탐색하며 패턴을 찾거나, 특정 조건에 맞는 위치를 기록하거나, 염기 빈도를 계산하는 일이 매우 흔합니다.

## G와 C의 개수 세기

이제 조건문과 반복문을 함께 사용해 보겠습니다. DNA 서열을 처음부터 끝까지 보면서 G와 C가 몇 번 등장하는지 세는 프로그램을 작성할 수 있습니다.

```python
dna = "ATGCCGTAGC"
g_count = 0
c_count = 0

for base in dna:
    if base == "G":
        g_count += 1
    elif base == "C":
        c_count += 1

print("G의 개수:", g_count)
print("C의 개수:", c_count)
```

이 코드에서 `g_count`와 `c_count`는 각각 G와 C의 개수를 저장하는 변수입니다. 반복문이 서열을 한 글자씩 읽고, 조건문이 현재 문자가 G인지 C인지 판단합니다. 조건이 참일 때마다 해당 개수를 1씩 늘립니다.

이 방식은 매우 중요합니다. 왜냐하면 앞으로 배울 GC 함량 계산, 상보 서열 생성, 패턴 검색 등도 모두 "하나씩 읽고 조건에 따라 처리하는 방식"에 기반하기 때문입니다.

## 예제 실습: DNA 유효성 검사기 만들기

이제 조건문과 반복문을 결합해 간단한 DNA 유효성 검사기를 만들어 보겠습니다.

```python
test_seq = "ATGCXTA"
is_valid = True

for base in test_seq:
    if base not in "ATGC":
        print(f"알림: '{base}'는 유효한 DNA 염기가 아닙니다.")
        is_valid = False
        break

if is_valid:
    print("이 서열은 정상적인 DNA 서열입니다.")
else:
    print("이 서열에는 잘못된 문자가 포함되어 있습니다.")
```

이 코드의 핵심은 다음과 같습니다.

1. 서열을 처음부터 끝까지 순회한다.
2. 각 문자가 A, T, G, C 중 하나인지 검사한다.
3. 잘못된 문자를 발견하면 경고 메시지를 출력한다.
4. 문제가 없으면 정상적인 DNA 서열이라고 판단한다.

실제로 연구 데이터를 다룰 때도 이런 검사는 매우 유용합니다. 작은 예제처럼 보이지만, 데이터 분석의 신뢰성을 지키는 중요한 첫 단계입니다.

## 핵심 정리

- 조건문은 데이터가 특정 조건을 만족하는지 판단할 때 사용한다.
- 반복문은 DNA 서열 전체를 처음부터 끝까지 순회할 때 사용한다.
- DNA 유효성 검사는 잘못된 데이터를 분석에 사용하지 않도록 막아 준다.
- 조건문과 반복문을 함께 사용하면 특정 염기의 개수를 셀 수 있다.
