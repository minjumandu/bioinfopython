# 1장 생물정보학과 파이썬의 첫 만남

DNA를 처음 배우면 많은 학생이 이렇게 느낍니다. "A, T, G, C 네 글자가 반복될 뿐인데, 왜 이것이 생명의 비밀이라고 할까?" 그런데 시선을 조금만 바꾸면 DNA는 꽤 익숙한 대상이 됩니다. 프로그래머에게 DNA는 긴 코드 문자열처럼 보이고, 데이터 분석가에게는 정교하게 정렬된 생명 데이터베이스처럼 보입니다. 세포는 이 정보를 읽고, 복사하고, 필요한 부분을 사용해 단백질을 만들어 냅니다. 마치 컴퓨터가 소스 코드를 해석하고 실행하듯이 말입니다.

이 관점은 생물정보학의 출발점입니다. 생명 현상을 신비로운 대상으로만 바라보는 대신, 저장되고 분석될 수 있는 정보로 이해하는 것입니다. 이렇게 보면 DNA 서열의 길이를 구하는 일, 특정 염기 패턴을 찾는 일, 종 사이의 유전적 차이를 비교하는 일이 모두 데이터 처리 문제로 바뀝니다. 그리고 이러한 작업을 가장 쉽고 유연하게 시작하게 해 주는 언어가 바로 파이썬입니다.

- 생물정보학 입문 언어로서 파이썬의 장점을 설명할 수 있다.
- 파이썬 문자열로 DNA 서열을 표현하고 길이를 구할 수 있다.

## 생물정보학이란 무엇인가?

생물정보학(Bioinformatics)은 생명과학 데이터를 컴퓨터로 저장하고, 처리하고, 해석하는 학문입니다. 여기서 말하는 생명과학 데이터에는 DNA 서열, RNA 발현량, 단백질 서열, 단백질 구조, 유전자 기능 정보, 질병 관련 변이 정보 등이 포함됩니다. 현대 생명과학은 실험만으로 이해하기 어려울 정도로 많은 데이터를 생산하므로, 컴퓨터를 이용한 분석이 필수적입니다.

생물정보학의 핵심은 단순히 데이터를 모아 두는 것이 아니라, 그 안에서 의미 있는 패턴을 찾아내는 데 있습니다. 예를 들어 다음과 같은 질문은 모두 생물정보학의 주제가 됩니다.

- 특정 유전자는 어떤 생물에서 얼마나 비슷한가?
- 질병이 있는 환자군과 없는 환자군의 유전체에는 어떤 차이가 있는가?
- 어떤 DNA 구간이 실제로 단백질을 만드는 유전자인가?
- 새로운 바이러스의 유전체는 기존 바이러스와 얼마나 유사한가?

이 질문들은 생물학적이지만, 해결 과정에는 계산이 필요합니다. 즉, 생물정보학은 생명과학의 질문을 컴퓨터가 다룰 수 있는 문제로 바꾸는 학문이라고 할 수 있습니다. 이런 점에서 생물정보학은 생명과학, 컴퓨터 과학, 통계학이 만나는 학제적 분야입니다.

특히 차세대 염기서열 분석 기술이 보편화된 오늘날에는 DNA를 읽는 속도가 매우 빨라졌습니다. 문제는 데이터를 얻는 것이 아니라, 얻은 데이터를 어떻게 해석하느냐에 있습니다. 따라서 생물정보학은 현대 생명과학 연구의 중심 도구가 되었습니다.

## 왜 수많은 언어 중 파이썬을 사용하는가?

생물정보학에서 프로그래밍 언어는 실험실의 장비와 비슷합니다. 어떤 장비를 선택하느냐에 따라 작업의 효율과 정확성이 달라지듯, 어떤 언어를 선택하느냐에 따라 분석의 난이도도 크게 달라집니다. 파이썬은 입문자와 연구자 모두에게 매우 적합한 언어입니다.

첫째, 문법이 간결하고 읽기 쉽습니다. 파이썬은 복잡한 기호보다 들여쓰기와 명확한 문장 구조를 활용하므로, 프로그래밍 경험이 적은 학습자도 코드를 비교적 쉽게 이해할 수 있습니다. 생물정보학 입문에서는 "데이터를 다루는 사고"가 중요하므로, 언어 자체가 너무 어려우면 학습이 지연됩니다. 파이썬은 이러한 부담을 줄여 줍니다.

둘째, 과학 계산과 데이터 분석을 위한 라이브러리가 풍부합니다. 배열 계산을 위한 `NumPy`, 표 형식 데이터 처리를 위한 `pandas`, 시각화를 위한 `matplotlib`, 생물정보학 전용 도구 모음인 `Biopython` 등은 파이썬이 연구 현장에서 널리 사용되는 중요한 이유입니다. 즉, 파이썬은 단순한 교육용 언어가 아니라 실제 연구에 바로 연결되는 실용적 언어입니다.

셋째, 텍스트 처리에 강합니다. 생물정보학 데이터는 FASTA, FASTQ, GFF 같은 텍스트 기반 형식으로 자주 저장됩니다. DNA 서열 역시 결국 문자들의 나열입니다. 따라서 문자열 처리 기능이 강력한 파이썬은 생물정보학과 매우 잘 어울립니다.

넷째, 확장성이 높습니다. 파이썬은 파일 읽기, 데이터 정리, 통계 처리, 시각화, 머신러닝, 자동화까지 하나의 흐름으로 연결할 수 있습니다. 즉, 작은 실습에서 시작해 실제 프로젝트로 확장하기에 적합합니다.

정리하면, 파이썬은 배우기 쉽고, 과학 데이터 분석에 강하며, 생물정보학 자료 형식과 잘 맞고, 연구 업무 전체를 연결하기 쉬운 언어입니다. 그래서 수많은 언어 중에서도 생물정보학 입문의 첫 언어로 자주 선택됩니다.

## 파이썬으로 DNA 서열의 길이를 구하는 아주 간단한 맛보기 코드

이제 가장 간단한 예제를 살펴보겠습니다. DNA 서열은 파이썬에서 문자열(`string`)로 표현할 수 있습니다. 문자열의 길이는 `len()` 함수로 구합니다.

```python
dna = "ATGCGTACGTTAGC"
length = len(dna)

print("DNA 서열:", dna)
print("서열의 길이:", length)
```

이 코드는 먼저 `dna` 변수에 DNA 서열을 저장한 뒤, `len(dna)`를 이용해 서열의 길이를 계산합니다. DNA의 각 염기 A, T, G, C는 문자 하나로 표현되므로, 문자열 길이는 곧 서열 길이와 같습니다.

실행 결과는 다음과 같습니다.

```text
DNA 서열: ATGCGTACGTTAGC
서열의 길이: 14
```

이 예제는 매우 단순하지만 중요한 의미를 가집니다. 컴퓨터는 DNA를 특별한 대상이 아니라 분석 가능한 문자열 데이터로 다룹니다. 따라서 문자열 처리의 기본 기능만 익혀도 생물정보학 분석의 첫걸음을 시작할 수 있습니다.

다음과 같이 직접 서열을 바꾸어 보면서 연습해 보는 것도 좋습니다.

```python
dna = "AATTGGCC"
print(len(dna))
```

이처럼 작은 실습을 통해 학습자는 자연스럽게 다음 질문으로 나아갈 수 있습니다. 특정 염기는 몇 번 나타나는가? GC 비율은 얼마인가? 두 서열은 얼마나 비슷한가? 생물정보학은 바로 이러한 질문을 코드로 바꾸는 과정입니다.

## 핵심 정리

- 생물정보학은 생명과학 데이터를 컴퓨터로 분석하는 학문이다.
- DNA는 생명의 정보를 담은 문자열 데이터로 볼 수 있다.
- 파이썬은 배우기 쉽고 생물정보학 분석 도구가 풍부하여 입문에 적합하다.
- `len()` 함수를 사용하면 DNA 서열의 길이를 쉽게 구할 수 있다.
