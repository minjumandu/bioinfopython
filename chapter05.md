# 5장 리스트와 딕셔너리로 DNA를 조작하기

지금까지 우리는 DNA 서열을 주로 문자열로 다루었습니다. 문자열은 DNA를 표현하기에 매우 편리하지만, 모든 상황에서 가장 유연한 자료형은 아닙니다. 예를 들어 서열의 특정 위치를 바꾸거나, 여러 개의 서열을 한꺼번에 관리하거나, 염기 간의 대응 관계를 체계적으로 나타내고 싶을 때는 다른 자료구조가 더 적합할 수 있습니다.

이 장에서는 리스트와 딕셔너리를 이용해 DNA 데이터를 보다 적극적으로 조작하는 방법을 배웁니다. 리스트는 순서가 있는 여러 값을 저장할 때 유용하고, 딕셔너리는 서로 대응되는 정보를 짝지어 관리할 때 강력합니다. 생물정보학에서는 이 두 자료구조가 매우 자주 등장합니다.

![그림 5-1. 리스트와 딕셔너리로 DNA 데이터를 조작하는 방식](figures/chapter05_structures.svg)

그림 5-1은 이 장의 핵심을 보여 줍니다. DNA 서열은 리스트로 바꾸어 수정할 수 있고, 상보 염기쌍은 딕셔너리로 표현할 수 있습니다. 즉, 자료구조를 바꾸면 데이터 처리 방식도 더 풍부해집니다.

## 학습 목표

- 문자열과 리스트의 차이를 설명할 수 있다.
- 리스트를 이용해 DNA 서열의 일부를 수정할 수 있다.
- 딕셔너리를 이용해 상보적 염기 관계를 표현할 수 있다.
- 상보 서열을 생성하는 간단한 프로그램을 작성할 수 있다.

## 문자열과 리스트의 차이

문자열은 문자들의 순서 있는 나열입니다. DNA 서열을 표현하기에 적합하지만, 문자열은 한 번 만들어지면 개별 문자를 직접 바꾸기 어렵습니다. 반면 리스트는 항목을 자유롭게 수정하거나 추가할 수 있습니다.

다음 코드를 비교해 보겠습니다.

```python
dna_str = "ATGCGT"
dna_list = list(dna_str)

print(dna_str)
print(dna_list)
```

실행하면 문자열은 `"ATGCGT"`처럼 보이고, 리스트는 `['A', 'T', 'G', 'C', 'G', 'T']`처럼 각 문자가 분리된 형태로 저장됩니다.

## 리스트로 염기서열 수정하기

리스트의 큰 장점은 특정 위치의 값을 바꿀 수 있다는 것입니다. 이를 이용하면 간단한 돌연변이 시뮬레이션도 해 볼 수 있습니다.

```python
dna_list = ['A', 'T', 'G', 'C', 'G', 'T']
dna_list[2] = 'A'

print(dna_list)
```

이 코드는 세 번째 염기인 `'G'`를 `'A'`로 바꿉니다. 문자열이었다면 이런 직접 수정이 불가능하지만, 리스트에서는 간단히 처리할 수 있습니다.

리스트에 새로운 염기를 추가하는 것도 가능합니다.

```python
dna_list.append('C')
print(dna_list)
```

이처럼 리스트는 DNA 서열을 더 유연하게 다루는 데 유용합니다.

## 딕셔너리로 상보적 염기쌍 표현하기

DNA의 중요한 특징 중 하나는 상보적 결합입니다. A는 T와, G는 C와 짝을 이룹니다. 이러한 관계는 딕셔너리로 표현하기에 매우 적합합니다.

```python
complement_map = {
    "A": "T",
    "T": "A",
    "G": "C",
    "C": "G"
}

print(complement_map["A"])
print(complement_map["G"])
```

딕셔너리는 "어떤 값에 대응하는 다른 값"을 빠르게 찾을 수 있게 해 줍니다. 생물정보학에서는 염기 상보 관계뿐 아니라, 유전자 이름과 기능, 샘플 번호와 측정값 같은 다양한 대응 관계를 표현하는 데 자주 사용됩니다.

## 상보 서열 생성하기

이제 딕셔너리와 반복문을 함께 사용해 상보 서열을 만들어 보겠습니다.

```python
def make_complement(dna_seq):
    complement_map = {
        "A": "T",
        "T": "A",
        "G": "C",
        "C": "G"
    }

    comp_seq = ""

    for base in dna_seq:
        comp_seq += complement_map[base]

    return comp_seq


print(make_complement("ATGC"))
```

이 함수는 DNA 서열을 한 글자씩 읽으면서, 각 염기를 상보적인 염기로 바꾸어 새로운 문자열을 만듭니다. `"ATGC"`를 입력하면 `"TACG"`가 출력됩니다.

## 예제 실습: DNA 바코드와 상보 서열 생성기

이제 리스트와 딕셔너리를 함께 활용해 간단한 실습을 해 보겠습니다.

```python
dna = "ATGCGT"
dna_list = list(dna)

# 돌연변이처럼 세 번째 염기를 바꾸기
dna_list[2] = "A"
mutated_dna = "".join(dna_list)

complement_map = {
    "A": "T",
    "T": "A",
    "G": "C",
    "C": "G"
}

complement = ""
for base in mutated_dna:
    complement += complement_map[base]

print("원래 서열:", dna)
print("변이 후 서열:", mutated_dna)
print("상보 서열:", complement)
```

이 예제는 세 가지 중요한 과정을 보여 줍니다.

1. 문자열을 리스트로 바꾼다.
2. 리스트의 값을 수정한다.
3. 딕셔너리를 이용해 상보 서열을 생성한다.

이러한 흐름은 이후 더 복잡한 생물정보학 데이터 조작의 기초가 됩니다.

## 핵심 정리

- 문자열은 읽기에는 편리하지만 수정에는 제약이 있다.
- 리스트는 여러 항목을 순서대로 저장하고 수정할 수 있다.
- 딕셔너리는 서로 대응되는 정보를 표현하는 데 적합하다.
- 상보적 염기쌍은 딕셔너리로 표현하면 편리하다.
