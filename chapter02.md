# 2장 변수와 자료형으로 다루는 DNA 서열

1장에서 우리는 DNA를 생명의 정보이자 분석 가능한 데이터로 바라보았습니다. 이제 한 걸음 더 나아가 보겠습니다. 데이터를 분석하려면 먼저 컴퓨터 안에 잘 저장해야 합니다. 생물정보학에서도 마찬가지입니다. DNA 서열, 유전자 이름, 샘플 번호, 실험 결과를 컴퓨터가 이해할 수 있는 형태로 담아 두어야 비로소 계산이 시작됩니다.

파이썬은 이 첫 단계를 매우 쉽게 만들어 줍니다. DNA 서열 하나는 문자열로 저장할 수 있고, 여러 개의 서열은 리스트에 담을 수 있으며, 이름표가 붙은 정보는 변수에 넣어 관리할 수 있습니다. 이 장에서는 파이썬의 가장 기본적인 개념인 변수와 자료형을 배우면서, 그것이 생물정보학 데이터와 어떻게 연결되는지 자연스럽게 익혀 보겠습니다.

![그림 2-1. 변수와 자료형으로 생물정보학 데이터를 저장하는 예](figures/chapter02_variables_datatypes.svg)

그림 2-1은 이 장의 핵심을 요약합니다. 하나의 DNA 서열은 문자열로, 여러 개의 서열은 리스트로, 길이와 같은 계산 결과는 숫자 자료형으로 저장할 수 있습니다. 즉, 생물정보학 데이터 분석의 출발점은 "알맞은 자료형에 데이터를 담는 일"입니다.

## 학습 목표

이 장을 읽고 나면 다음 내용을 설명하고 실습할 수 있어야 합니다.

- 변수의 의미를 설명하고 값을 저장할 수 있다.
- 문자열, 정수, 리스트의 기본 개념을 이해할 수 있다.
- DNA 서열을 문자열로 저장하고 길이를 계산할 수 있다.
- 여러 개의 DNA 서열을 리스트로 저장하고 원하는 값을 꺼낼 수 있다.

## 변수란 무엇인가?

변수(variable)는 데이터를 저장하는 이름표가 붙은 상자라고 생각하면 이해하기 쉽습니다. 예를 들어 실험실에서 시료관에 라벨을 붙여 샘플을 구분하듯, 파이썬에서는 변수 이름을 사용해 데이터를 구분합니다.

다음 코드를 살펴보겠습니다.

```python
dna = "ATGCGTAC"
sample_name = "Patient_01"
length = 8
```

여기서 `dna`, `sample_name`, `length`는 모두 변수 이름입니다. 각 변수는 서로 다른 종류의 데이터를 저장합니다. `dna`에는 DNA 서열이, `sample_name`에는 샘플 이름이, `length`에는 숫자 값이 들어 있습니다.

변수는 데이터를 다시 사용할 수 있게 해 준다는 점에서 매우 중요합니다. 예를 들어 DNA 서열을 변수에 저장해 두면, 길이를 구하거나, 특정 염기를 세거나, 일부를 잘라 보는 작업을 반복해서 수행할 수 있습니다.

```python
dna = "ATGCGTAC"
print(dna)
print(len(dna))
```

이처럼 변수는 단순히 값을 저장하는 기능을 넘어, 분석 과정을 읽기 쉽고 체계적으로 만드는 역할도 합니다.

## 문자열과 DNA 서열

파이썬에서 문자열(`string`)은 문자들의 나열입니다. DNA 서열은 A, T, G, C라는 문자로 이루어져 있으므로 문자열 자료형으로 표현하기에 매우 적합합니다.

```python
dna = "AATGCCGT"
print(dna)
print(type(dna))
```

이 코드에서 `dna`는 문자열입니다. 문자열이라는 점이 중요한 이유는, 파이썬이 문자열을 다루기 위한 다양한 기능을 제공하기 때문입니다. 생물정보학에서는 이 기능을 이용해 서열의 길이를 구하고, 특정 위치의 염기를 확인하고, 부분 서열을 추출할 수 있습니다.

예를 들어 다음과 같이 서열의 길이와 첫 번째 염기를 확인할 수 있습니다.

```python
dna = "AATGCCGT"

print("서열의 길이:", len(dna))
print("첫 번째 염기:", dna[0])
print("세 번째 염기:", dna[2])
```

파이썬에서는 문자열의 위치를 셀 때 0부터 시작합니다. 따라서 `dna[0]`은 첫 번째 문자, `dna[1]`은 두 번째 문자, `dna[2]`는 세 번째 문자를 뜻합니다. 처음에는 다소 낯설 수 있지만, 이후 서열 분석에서 매우 자주 사용되는 규칙이므로 익숙해져야 합니다.

부분 서열을 잘라 내는 것도 가능합니다.

```python
dna = "AATGCCGT"
print(dna[0:4])
```

실행 결과는 `AATG`입니다. 즉, 처음 네 개의 염기를 잘라 낸 것입니다. 이와 같은 슬라이싱(slicing)은 특정 구간의 서열을 분석할 때 매우 유용합니다.

## 숫자 자료형과 서열 길이

생물정보학에서는 문자열만 다루는 것이 아닙니다. 서열 길이, 염기 개수, GC 비율, 샘플 수처럼 숫자 데이터도 자주 다룹니다. 파이썬에서는 정수(`int`)와 실수(`float`) 같은 숫자 자료형을 제공합니다.

```python
dna = "AATGCCGT"
length = len(dna)
gc_count = 4
gc_ratio = gc_count / length

print(length)
print(gc_ratio)
```

위 예제에서 `length`와 `gc_count`는 정수이고, `gc_ratio`는 나눗셈 결과이므로 실수입니다. 이런 숫자 자료형은 이후 통계 처리나 그래프 작성에서도 계속 사용됩니다.

중요한 점은, 서로 다른 자료형이 각기 다른 목적에 맞게 사용된다는 사실입니다. DNA 자체는 문자열로 저장하고, 길이나 개수는 숫자로 저장하는 식입니다. 자료형을 올바르게 선택하면 프로그램이 더 명확해지고 오류도 줄어듭니다.

## 리스트와 여러 개의 DNA 서열

실제 생물정보학에서는 하나의 서열만 다루는 경우보다 여러 개의 서열을 함께 다루는 경우가 훨씬 많습니다. 예를 들어 여러 샘플의 DNA 조각을 비교하거나, 여러 종의 유전자 서열을 나란히 분석할 수 있습니다. 이때 사용할 수 있는 자료형이 리스트(`list`)입니다.

리스트는 여러 값을 순서대로 저장하는 자료형입니다.

```python
sequences = ["ATGC", "AATTGG", "CCGTA", "GGGCCC"]
print(sequences)
```

리스트 안에는 문자열, 숫자, 심지어 또 다른 리스트도 들어갈 수 있습니다. 생물정보학 입문에서는 여러 DNA 서열을 한데 모아 두는 용도로 리스트를 자주 사용합니다.

리스트의 각 값에도 위치 번호가 있습니다.

```python
sequences = ["ATGC", "AATTGG", "CCGTA", "GGGCCC"]

print("첫 번째 서열:", sequences[0])
print("두 번째 서열:", sequences[1])
print("리스트의 길이:", len(sequences))
```

이 예제에서 `sequences[0]`은 첫 번째 DNA 서열이고, `len(sequences)`는 리스트 안에 들어 있는 서열의 개수입니다. 즉, `len()` 함수는 문자열에도 사용할 수 있고 리스트에도 사용할 수 있습니다. 다만 문자열에 쓰면 문자 수를, 리스트에 쓰면 항목 수를 반환합니다.

## 예제 실습: 샘플 서열 정보 저장하기

이제 지금까지 배운 내용을 바탕으로 간단한 실습을 해 보겠습니다.

```python
sample_name = "Sample_A"
dna = "ATGCGTAACG"
length = len(dna)

print("샘플 이름:", sample_name)
print("DNA 서열:", dna)
print("서열 길이:", length)
print("앞 5개 염기:", dna[:5])
```

이 코드는 하나의 샘플에 대해 이름, DNA 서열, 서열 길이, 그리고 앞부분 일부 서열을 출력합니다. 비록 짧은 예제이지만, 실제 생물정보학 분석의 기본 흐름이 이미 들어 있습니다.

1. 데이터를 변수에 저장한다.
2. 필요한 값을 계산한다.
3. 결과를 확인한다.

이러한 흐름은 앞으로 파일 읽기, 염기 수 세기, GC 비율 계산, 여러 서열 비교 같은 더 복잡한 주제로 자연스럽게 이어질 것입니다.

## 핵심 정리

- 변수는 데이터를 저장하는 이름표가 붙은 상자와 같다.
- DNA 서열은 문자열 자료형으로 표현할 수 있다.
- `len()` 함수는 문자열의 길이와 리스트의 항목 수를 구할 수 있다.
- 숫자 자료형은 길이, 개수, 비율 같은 계산 결과를 저장할 때 사용한다.
- 리스트는 여러 개의 DNA 서열을 한 번에 저장할 때 유용하다.
